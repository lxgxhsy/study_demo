# OpenSpec Project Notes

## Purpose

Concurrency Lab is a small backend lab for dynamic thread pool behavior, Leaf-style distributed ID generation, and pressure-test learning.

## Governance Rule

Use OpenSpec for behavior-changing work:

1. Create or update a change under `openspec/changes/<change-id>/`.
2. Keep requirements in spec deltas.
3. Keep implementation tasks in `tasks.md`.
4. Implement only after the change is understandable enough to verify.
